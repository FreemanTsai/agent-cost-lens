# Agent Cost Lens

[![License: MIT](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![Node.js](https://img.shields.io/badge/node-%3E%3D18-brightgreen)](.nvmrc)

Analyze Codex / Code‑based AI agent session logs (`*.jsonl`) — visualize token usage, estimate API costs, and surface optimization opportunities.

---

## Features

- **Parsing engine** — reads Codex session logs from `~/.codex/sessions/`, aggregates by day, project, session, turn, and step
- **Dashboard** — a single HTML page with three views:
  - **Stats** — cost, token volumes, cache rates grouped by project, model, tool, skill, and session
  - **Search** — full‑text search across sessions, with session detail drill‑down (turns, steps, sub‑agents)
  - **Optimize** — Cost Lens scoring for sessions with likely token burn
- **Monitor** — live dashboard for current rate limits, spend, cache efficiency, and latest sessions
- **Cost estimation** — per‑model pricing (GPT‑5.5 / 5.4 / 5.4 Mini), accounts for cached vs uncached input tokens
- **No database** — works entirely with static JSON files; no server‑side storage needed

## Getting Started

### Prerequisites

- Node.js ≥ 18
- Codex session logs at `~/.codex/sessions/` (auto‑generated by Codex Desktop or CLI)

### Run

```bash
git clone https://github.com/FreemanTsai/agent-cost-lens.git
cd agent-cost-lens
bash start.sh
```

`start.sh` parses recent logs, starts the server on `0.0.0.0:8080`, and opens the dashboard. Other devices on the same network can connect with your machine's LAN IP:

```bash
ipconfig getifaddr en0
# then open http://<LAN-IP>:8080
```

Or step by step:

```bash
# Parse logs → generate daily JSON files + Cost Lens report
node scripts/parse-codex-logs.mjs

# Start the dev server
node scripts/server.mjs
```

Open [http://localhost:8080](http://localhost:8080) in your browser.

To bind the server manually:

```bash
HOST=0.0.0.0 PORT=8080 node scripts/server.mjs
```

### CLI Flags

```bash
node scripts/parse-codex-logs.mjs --date=2026-06-15    # Process a specific date only
node scripts/parse-codex-logs.mjs --date-only          # Skip optimize analysis, only produce daily JSON
node scripts/parse-codex-logs.mjs --all                # Process all available log files
```

- cache hit rate

## Cost Lens Analysis

The **Optimize** tab shows Cost Lens: a ranked list of sessions with likely token burn. Each session starts at `100`; detected reasons subtract points, and the dashboard shows the highest-priority matched reason as the **Primary Reason**.

| Priority | Primary Reason | Trigger | Points |
|---------:|----------------|---------|-------:|
| 1 | Repeated Rework | Consecutive verification retries without an edit or a different command | low 5, medium 10, high 20 |
| 2 | Search Heavy / Low Edit | `rg` + `sed` + `cat` >= 10 and search/edit ratio >= 10:1 | medium 6, high 12 |
| 3 | Test Heavy / Low Edit | test commands >= 5 and test/edit ratio >= 5:1 | medium 5, high 10 |
| 4 | Context Heavy | effective input tokens are high | low 1, medium 3, high 6 |
| 5 | High Verification Activity | build/lint/test commands >= 8 while test commands < 5 | medium 4, high 8 |

Generated reports are written to `public/data/codex-burn*.json`. `public/data/codex-optimize.json` and period-specific `public/data/codex-optimize*.json` files are kept as empty compatibility reports for the dashboard loader.

## Pricing

| Model          | Input ($/1M tok) | Cached ($/1M tok) | Output ($/1M tok) |
|----------------|------------------:|-------------------:|-------------------:|
| GPT‑5.5        |              5.00 |               0.50 |              30.00 |
| GPT‑5.4        |              2.50 |               0.25 |              15.00 |
| GPT‑5.4 Mini   |              0.75 |               0.075 |               4.50 |

Cache billing: `cache_writes = 1.25 × input_price`, `cache_reads = 0.10 × input_price`.

## Data Model

```
Project
 └─ Session              (one .jsonl file)
     └─ Turn             (one user message → one Codex response cycle)
         └─ Step         (single tool call / response)

Session
 ├─ projectName — basename of the working directory
 ├─ sessionId  — extracted from the rollout filename
 ├─ turns[]    — user→Codex turn groups
 ├─ costUsd    — accumulated per-step cost
 └─ tokens     — input, cachedInput, output, reasoningOutput
```

Daily JSON files are generated per day to avoid memory issues with large datasets.
